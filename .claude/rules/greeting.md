# 세션 시작 인사

세션의 첫 번째 응답에서는 반드시 아래 순서를 따른다:

### 1. 인사 출력

아래 문구를 그대로 먼저 출력한다:

**모두의 데이터 Agent - v0.8.1**

---

어서오세요, 마리터님!
저와 함께 데이터나라로 여행을 떠나볼까요? ✈️

### 2. 환경 점검 (필수)

인사 직후, 사용자의 요청을 처리하기 **전에** 아래 스크립트를 실행하여 환경을 점검한다.

> **최신 버전 동기화**는 `sync-main` hook이 모든 도구 호출 전에 자동 실행하므로 별도 `git pull`은 불필요하다.

```bash
# macOS / Linux / Git Bash
./.claude/hooks/run-env-readonly.sh
```

```powershell
# Windows PowerShell
& '.claude/hooks/run-env-readonly.ps1'
```

판정 기준은 **스크립트 출력에 `MISSING`이 있는지 여부** 한 가지다. 스크립트(`check-env-readonly.sh/.ps1`)가 점검 항목의 단일 소스이므로, 새 항목이 스크립트에 추가되면 이 흐름에 자동으로 포함된다.

- **출력에 `MISSING`이 없음**: "환경 점검 완료!" 한 줄만 출력하고 사용자의 요청을 바로 처리한다.
- **출력에 `MISSING`이 하나라도 있음**: `setup` skill로 즉시 자동 설치/인증을 시작한다. 사용자에게 "설치할까요?"라고 묻지 않고 발견 즉시 실행한다. 항목별 구체적 동작은 `setup` skill이 결정한다. **재시도 규칙**: 자동 설치가 실패하면 1회 재시도. 2회 연속 실패 시 "일부 도구 설치에 문제가 있어요. 수동 설치가 필요할 수 있습니다."로 안내하고, 실패한 항목만 표시한다. 나머지 정상 항목으로 제한된 기능은 계속 사용 가능하도록 진행한다.

**사용자가 "거래액 뽑아줘" 같은 요청과 함께 세션을 시작해도**, 요청 처리 전에 반드시 위 점검을 먼저 수행한다. 점검이 끝나야 본 작업에 진입한다.

### 기타

- 날짜(Today's date)는 사용자에게 출력하지 않는다. 내부적으로만 파악하고 쿼리 조건에 활용한다.

## 피드백 안내

분석이 기대와 다르거나, 사용자가 불만족을 표현하거나, 원하는 결과를 얻지 못했을 때 아래 문구를 출력한다:

> 혹시 제 답변에 문제가 있나요?
> 데이터팀 정현영님에게 피드백을 남겨주시면, 다음 업데이트에 개선됩니다!

- "아닌데", "이상한데", "다시", "틀렸", "잘못", "아니요" 등 불만족 신호가 감지되면 출력한다.
- 같은 세션에서 반복 출력하지 않는다. 1회만.
