# 개인 메모리 규칙 (빌트인 auto memory 사용)

Claude Code 빌트인 `autoMemoryEnabled` 기능으로 메모리를 관리한다. `.claude/MEMORY.md` 파일은 사용하지 않는다.

## 자동 수집 대상

대화 중 아래 항목이 등장하면, 빌트인 auto memory에 적절한 type으로 저장한다.

| 카테고리 | 감지 신호 | memory type |
|----------|----------|-------------|
| **자주 조회하는 대시보드** | Redash URL, Grafana URL, Looker URL 등 | `reference` |
| **자주 조회하는 Google Sheets** | Google Sheets URL, 스프레드시트 ID | `reference` |
| **자주 쓰는 지표/쿼리 패턴** | 같은 테이블+지표 조합이 2회 이상 요청됨 | `reference` |
| **자주 쓰는 Slack 채널** | Slack URL에서 채널 반복 등장 | `reference` |
| **자주 쓰는 Confluence 페이지** | Confluence URL 반복 등장 | `reference` |
| **자주 조회하는 Jira 프로젝트/보드** | Jira URL, 프로젝트 키 반복 | `reference` |
| **주요 업무 맥락** | 진행 중인 프로젝트, 반복 요청 주제 | `project` |
| **사용자 선호/피드백** | "이렇게 해줘", "이건 하지 마", 교정 | `feedback` |
| **🔴 명시적 기억 요청** | "기억해줘", "기억해둬", "메모해줘", "저장해둬", "잊지 마" | 내용에 맞는 type |
| **🔴 지시사항/규칙** | "앞으로 ~해줘", "~하지 마", "항상 ~", "매번 ~", "다음부터 ~" | `feedback` |
| **🔴 사실/정의 교정** | "그게 아니라 ~야", "~가 맞아", "~는 ~를 의미해" | `project` |

> 🔴 표시 항목은 **최우선 저장 대상**이다. 다른 항목은 반복/패턴 감지가 필요하지만, 🔴 항목은 **1회 등장만으로 즉시** 저장한다.

## 저장 규칙

- **중복 방지**: 이미 같은 내용의 memory가 있으면 추가하지 않는다. 설명이 더 구체화되었으면 기존 항목을 업데이트한다.
- **저장 후 확인 응답**: 저장할 때 **"이 내용은 제가 잘 기억해둘게요!"** 라고 한 줄 응답한다. 단, 본 작업 흐름을 끊지 않도록 응답 말미나 자연스러운 위치에 배치한다.
