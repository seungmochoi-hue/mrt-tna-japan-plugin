# 검색 Skill 공통 템플릿

모든 search skill(`search-all`, `search-slack`, `search-confluence`, `search-jira`, `search-gsheets`)이 공유하는 절차.

## 검색어 생성

사용자 요청에서 핵심 주제를 파악하고, 아래 축으로 검색어를 **최소 3개, 최대 6개** 만든다.

| 축 | 예시 |
|------|------|
| 원래 키워드 | `reservation_travelers` |
| 한국어 번역/동의어 | `여행자 정보 테이블`, `예약 인원` |
| 관련 테이블/컬럼명 | `RESVE_PRSNL_CNT`, `traveler_age_type` |
| 비즈니스 맥락 | `패키지 인원 과대 집계`, `마이팩 10인 이상` |

검색어 중 소스 전용을 구분할 필요 없다. 같은 검색어를 모든 소스에 쓴다.

## 대용량 결과 처리

검색 결과가 tool result 크기를 초과하면 파일로 저장된다. 이 경우:

1. 저장된 파일 경로를 확인한다
2. `Grep`으로 파일 내에서 핵심 필드(`Permalink`, `url` 등)를 검색하여 링크를 추출한다
3. 필요한 항목만 `Read`로 offset/limit를 조절하여 읽는다

## 결과 정리 공통 규칙

- **중복 제거**: 같은 URL을 가진 결과는 하나만 유지한다
- **관련도 순 정렬**: 검색 주제와 관련도가 높은 순서로 배치한다
- 한국어로 결과를 정리한다
- 결과가 없으면 "관련 내용을 찾지 못했어요"로 안내하고, 추가 검색어를 제안한다
- 사용자가 추가 검색을 요청하면 새로운 키워드로 검색어 생성부터 반복한다
