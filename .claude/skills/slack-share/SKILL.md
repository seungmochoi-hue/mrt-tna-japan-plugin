# Slack Share

분석 결과를 Slack 채널에 공유하는 skill.

## 핵심 원칙

긴 내용을 채널에 직접 올리지 않는다. 항상 스레드 안쪽으로만 공유한다.

```
공유 방식 선택
    │
    ├─ [1] 새 글 작성 ──── 제목 메시지를 채널에 올리고, 본문은 스레드에 작성
    │
    └─ [2] 기존 스레드에 답글 ── 이미 있는 스레드에 내용만 추가
```

## 절차

### Step 1 -- 공유 방식 선택

아래 옵션을 일반 텍스트로 제시한다.

Slack에 어떻게 공유할까요?

**1.** 새 글 작성 -- 제목을 채널에 올리고, 상세 내용은 스레드에 작성
**2.** 기존 스레드에 답글 -- 이미 있는 대화 스레드에 내용만 추가

### Step 2 -- 채널/스레드 지정

- **1 선택 시**: "어느 채널에 공유할까요? (채널명 또는 검색 키워드)" 질문. `slack_search_channels`로 채널을 찾아 확인한다.
- **2 선택 시**: "답글을 달 스레드의 Slack 링크를 알려주세요!" 질문. 링크에서 channel_id와 thread_ts를 추출한다.

> **Slack MCP 도구**: 이 skill은 `mcp__plugin_slack_slack__*` 도구를 사용한다 (`mcp__slack-bot__*`가 아님). 채널 검색: `mcp__plugin_slack_slack__slack_search_channels`, 메시지 전송: `mcp__plugin_slack_slack__slack_send_message`.

### Step 3 -- 제목 작성 (1번만 해당)

- 세션 분석 내용을 바탕으로 제목 초안을 제안한다.
- 제목은 1줄로 작성한다.
- 사용자가 수정하면 반영하고, "좋아요" / "진행" 등 확인하면 다음 단계로.

### Step 4 -- 본문 작성

- 세션 분석 내용을 바탕으로 본문 초안을 제안한다.
- 본문은 Slack mrkdwn 형식으로 작성한다. (마크다운이 아닌 Slack mrkdwn 문법 사용: `` `code` ``, `` ``` ``코드블록`` ``` ``, `>` 인용, `• ` bullet)
- 핵심 수치·결론·키워드는 backtick(`` `code` ``)으로, 항목 나열은 bullet(`• `)으로 구조화하여 가독성을 높인다.
- 본문 길이가 5,000자를 초과하면 여러 스레드 답글로 분할한다.
- 사용자가 수정하면 반영하고, 확인하면 다음 단계로.

### Step 5 -- 미리보기 및 즉시 전송

- 제목과 본문을 미리보기로 보여주고, 확인 질문 없이 바로 전송한다.
- 전송 순서 (1번의 경우):
  1. `slack_send_message`로 제목 메시지 전송 → `message_ts` 확보
  2. 확보된 `message_ts`를 `thread_ts`로 사용하여 본문을 스레드 답글로 전송
- 전송 순서 (2번의 경우):
  1. 지정된 `thread_ts`로 본문을 스레드 답글로 전송
- 전송 완료 후 Slack 메시지 링크를 사용자에게 안내한다.

## 데이터 공유 (Google Sheets 연계)

세션에서 Google Sheets에 내보낸 데이터를 Slack에 공유할 때는, 스프레드시트 링크를 스레드에 포함한다.

### 공유 방식 (자동 판단)

| 조건 | 방식 |
|------|------|
| 50행 이하 | 스레드에 코드블록으로 데이터 직접 전송 + Sheets 링크 |
| 50행 초과 | 핵심 요약 + 샘플 데이터만 스레드에 전송 + 전체 Sheets 링크 |

## Slack mrkdwn 규칙

- **`*bold*`, `_italic_` 사용 금지**: 강조는 backtick(`` ` ``) 코드 포맷만 사용한다. `_`가 포함된 테이블명·컬럼명은 반드시 backtick으로 감싼다.
- **불릿**: `• ` 1단계만 사용. 중첩 불릿(`◦`) 사용 금지 (Slack이 들여쓰기를 무시하여 같은 레벨로 렌더링됨).
- **섹션 구분**: `> ` (blockquote) + backtick으로 키워드 강조.
- 본문에 표 데이터가 있으면 Slack mrkdwn 코드블록으로 감싸 정렬을 유지한다.
- **코드블록 내 backtick 이스케이핑**: 데이터나 쿼리에 `` ``` `` (triple backtick)이 포함된 경우, `'''`(작은따옴표 3개)로 치환하거나 코드블록을 분할한다.
- 코드블록 표의 한글/영문 헤더 규칙은 `response-format.md`의 "Slack 코드블록 표 정렬 규칙" 참조.

## 인증 에러 처리

MCP 인증 에러 시 `.claude/rules/mcp-auth-recovery.md`의 절차를 따른다.

## 규칙

- 채널에 직접 올리는 메시지는 제목(1줄)만 허용한다. 본문/상세 내용은 반드시 스레드로 전송한다.
- 미리보기를 보여준 뒤 바로 전송한다. 별도 확인 질문 없이 진행한다.
- 한국어 안내 메시지 사용.
