# 회의록 Action Items → Slack DM 발송

Caret **사내미팅** 폴더에서 오늘 날짜의 회의록을 읽어 `🚀 Next Action Items` 섹션을 추출하고, 본인 Slack DM으로 전송한다.

## 실행 순서

### 1단계: 오늘 날짜 기준 노트 조회

`caret_list_notes`로 **사내미팅** 폴더의 노트 목록을 가져온다.

- 오늘 날짜(`YYYY-MM-DD`)로 생성 또는 수정된 노트만 대상으로 한다.
- 노트가 없으면 → "오늘 사내미팅 폴더에 회의록이 없습니다." 출력 후 종료.

### 2단계: 각 회의록에서 액션 아이템 추출

각 노트에 대해 `caret_get_note`로 내용을 가져온 뒤, 아래 규칙으로 파싱한다.

**추출 규칙:**
- `🚀 Next Action Items` 헤딩 이후의 내용을 파싱
- 다음 헤딩(`##`, `###`, `🚀`, `✅` 등 다른 섹션)이 나오면 종료
- 각 항목은 `- `, `• `, `[ ]`, `[x]` 형태의 리스트 항목
- 빈 줄만 있거나 항목이 없으면 해당 노트는 건너뜀

**내부 결과 구조:**
```
{
  note_title: "회의록 제목",
  note_id: "...",
  action_items: ["액션 아이템 1", "액션 아이템 2", ...]
}
```

### 3단계: 액션 아이템 없음 처리

모든 노트를 확인했는데 액션 아이템이 하나도 없으면:
> "오늘 사내미팅 회의록에 🚀 Next Action Items가 없습니다." 출력 후 종료.

### 4단계: Slack 메시지 구성

아래 형식으로 메시지를 만든다.

```
📋 *오늘의 회의 Action Items* — {YYYY-MM-DD}

*[회의록 제목 1]*
• 액션 아이템 1
• 액션 아이템 2

*[회의록 제목 2]*
• 액션 아이템 1

━━━━━━━━━━━━━━━━━
총 {N}개 미팅 · {M}개 액션 아이템
```

- 회의록이 1개면 제목 섹션 없이 액션 아이템만 나열해도 됨
- 완료된 항목(`[x]`)은 ~~취소선~~ 처리

### 5단계: Slack DM 발송

`mcp__plugin_slack_slack__slack_send_message` 툴로 본인에게 DM을 보낸다.

- **채널 ID**: 본인의 Slack DM 채널 ID (형식: `D` + 숫자)
- 처음 실행 시 Slack 앱 → 본인 프로필 → 멤버 ID로 확인 가능
- 또는 이전 DM 스레드의 채널 ID를 재사용

**발송 시 채널을 확인하지 않고 바로 전송한다** (자동화 흐름이므로 confirm 생략).

### 6단계: 완료 보고

터미널에 간단히 출력:
```
✅ Slack DM 발송 완료
   - 회의록 {N}건 처리
   - 액션 아이템 {M}개 전송
```

## 오류 처리

| 상황 | 대응 |
|------|------|
| Caret MCP 연결 실패 | 오류 출력 후 중단. `claude mcp list`로 연결 확인 안내 |
| 사내미팅 폴더 없음 | 폴더명 오타 가능성 안내 (대소문자·띄어쓰기 확인) |
| Slack 발송 실패 | 추출된 액션 아이템을 터미널에 출력하고 Slack 실패만 알림 |
| 액션 아이템 없음 | 조용히 종료 (자동 실행 시 불필요한 DM 방지) |
