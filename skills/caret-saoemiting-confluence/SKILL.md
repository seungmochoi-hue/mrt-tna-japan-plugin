# 사외미팅 회의록 → Confluence 페이지 자동 생성

Caret **사외미팅** 태그 노트를 읽어 Confluence 위키에 회의록 페이지를 생성한다.

- **부모 페이지 ID**: `5420023965` (TA 스페이스 내 `26` 폴더)
- **페이지 명 규칙**: `YYMMDD_파트너Name_대면 or 비대면_미팅`
  - 예: `260320_KKday_대면_미팅`, `260321_BicCamera_비대면_미팅`

---

## 날짜 결정 규칙

| 입력 예시 | 대상 날짜 |
|-----------|-----------|
| 날짜 언급 없음 / "오늘" | 당일 |
| "어제" | 전일 |
| "3월 20일", "2026-03-20" | 해당 날짜 |

대상 날짜를 `YYYY-MM-DD`로 확정한 뒤 진행한다.

---

## 실행 순서

### 1단계: 사외미팅 노트 조회

`mcp__caret__caret_list_notes`로 대상 날짜의 노트를 가져온다.

```
fromDate: {대상 날짜}T00:00:00
toDate:   {대상 날짜}T23:59:59
limit: 100
```

응답의 `tags` 배열에서 **name이 `사외미팅`인 노트만** 추린다.

- 해당 노트가 없으면 → `"{대상 날짜}에 사외미팅 태그 회의록이 없습니다."` 출력 후 종료.

### 2단계: 각 노트에서 메타 정보 추출

각 노트에 대해 `mcp__caret__caret_get_note`로 전문을 가져온 뒤, 아래 정보를 파싱한다.

#### 미팅 날짜 추출 (YYMMDD)
노트 **제목**에서 날짜를 추출한다. 다양한 형식을 허용한다.

| 제목 예시 | 추출 결과 |
|-----------|-----------|
| `260320 KKday 미팅` | `260320` |
| `2026-03-20 BicCamera 파트너 미팅` | `260320` |
| `KKday 미팅 (3/20)` | `260320` |

날짜를 못 찾으면 대상 날짜(`YYYY-MM-DD`)를 `YYMMDD` 형식으로 변환해 사용한다.

#### 파트너명 추출
노트 **제목**에서 날짜·공통 단어(`미팅`, `회의`, `파트너`, `방문`, `면담`)를 제거하고 남은 핵심 명칭을 파트너명으로 사용한다.

| 제목 예시 | 파트너명 |
|-----------|----------|
| `260320 KKday 미팅` | `KKday` |
| `2026-03-20 아라시투어 파트너 미팅` | `아라시투어` |
| `BicCamera 방문 260321` | `BicCamera` |

파트너명을 특정할 수 없으면 `Unknown`으로 표기한다.

#### 대면/비대면 판별
노트 **본문**에서 장소/방식 관련 섹션을 찾는다. 아래 키워드로 판별한다.

| 키워드 | 판별 결과 |
|--------|-----------|
| `대면`, `오프라인`, `현장`, `방문`, `직접` | **대면** |
| `비대면`, `온라인`, `화상`, `Zoom`, `줌`, `Teams`, `Google Meet`, `원격` | **비대면** |

두 키워드가 모두 없거나 모호하면 **대면**으로 기본 처리하고, 페이지 생성 후 터미널에 `⚠️ 대면/비대면 판별 불확실 — 확인 필요` 안내를 출력한다.

### 3단계: 중복 페이지 확인

생성하려는 페이지 제목으로 이미 Confluence에 페이지가 존재하는지 `mcp__claude_ai_Atlassian__searchConfluenceUsingCql`로 확인한다.

```
CQL: title = "{페이지명}" AND space = "TA" AND parent = 5420023965
```

이미 존재하면 → `"이미 페이지가 존재합니다: {페이지명} — 건너뜀"` 출력 후 해당 노트는 스킵.

### 4단계: Confluence 페이지 생성

`mcp__claude_ai_Atlassian__createConfluencePage`로 페이지를 만든다.

**파라미터:**
```
spaceKey:       "TA"
parentPageId:   "5420023965"
title:          "{YYMMDD}_{파트너명}_{대면 or 비대면}_미팅"
content:        노트 전문 (아래 변환 규칙 적용)
```

**Markdown → Confluence 변환 규칙:**
- `## 제목` → Confluence Heading 2
- `### 제목` → Confluence Heading 3
- `- 항목` → 불릿 리스트
- `[ ]` / `[x]` → 체크박스 (Confluence task list)
- 코드 블록(` ``` `) → Confluence code macro
- 나머지 텍스트는 그대로 단락으로 삽입

content는 Confluence Storage Format(XML)이 아닌 **wiki markup** 또는 **Markdown**으로 전달해도 MCP가 처리하므로, 노트 원문을 최대한 유지한다.

### 5단계: 완료 보고

터미널에 출력:
```
✅ Confluence 페이지 생성 완료
   - 처리 노트: {N}건
   - 생성된 페이지: {페이지명1}, {페이지명2}, ...
   - 스킵(중복): {M}건
   - URL: https://myrealtrip.atlassian.net/wiki/spaces/TA/pages/{생성된 페이지 ID}
```

## 오류 처리

| 상황 | 대응 |
|------|------|
| Caret MCP 연결 실패 | 오류 출력 후 중단. `claude mcp list`로 연결 확인 안내 |
| Confluence MCP 연결 실패 | 오류 출력 후 중단. Atlassian 인증 확인 안내 |
| 파트너명 추출 실패 | `Unknown` 사용 후 계속 진행, 터미널에 수동 수정 안내 출력 |
| 페이지 생성 실패 | 실패 이유 출력. 노트 내용은 터미널에 요약 출력 |
| 사외미팅 노트 없음 | 조용히 종료 (자동 실행 시 불필요한 출력 방지) |

## 자동 실행 정보

- 실행 시간: 매일 20:00
- CronJob으로 설정됨
- 직접 실행: "오늘 사외미팅 위키 올려줘"

### CronJob 설정 방법 (최초 1회)

```
/update-config 매일 20:00에 "오늘 사외미팅 Confluence 페이지 만들어줘" 실행
```
