# Confluence Q&A 스킬

Confluence 위키 페이지를 검색하고, 검색된 내용을 기반으로 한국어·일본어 병렬 답변을 제공한다.

## 실행 순서

### 1단계: 질문 및 Space 수집

`AskUserQuestion`으로 두 가지를 동시에 수집한다:
- 검색할 **Confluence Space key** (예: `~teamspace`, `DEV`, `HR`)
- 사용자의 **질문** (무엇을 알고 싶은지)

### 2단계: Confluence 검색

수집한 정보로 Atlassian MCP를 사용해 검색한다.

**우선 CQL 검색 사용:**
```
mcp__plugin_atlassian_atlassian__searchConfluenceUsingCql
  cql: 'space = "SPACE_KEY" AND text ~ "검색 키워드" ORDER BY lastmodified DESC'
  limit: 5
```

검색 결과가 없거나 부족하면 키워드를 분해해 재검색:
- 긴 질문 → 핵심 키워드 2~3개로 분리해 각각 검색
- 검색 결과가 있으면 `getConfluencePage`로 전체 본문 읽기

### 3단계: 답변 생성

검색된 페이지 내용을 분석해 질문에 맞는 답변을 구성한다.

**답변 형식 (한국어·일본어 병렬):**

---
## 질문 / 質問
> {사용자 질문}

---
## 답변 / 回答

**한국어**
{검색된 위키 내용 기반의 명확한 답변. 핵심만 간결하게.}

**日本語**
{위와 동일한 내용을 일본어로 번역}

---
## 출처 / 出典

| 페이지 제목 | リンク |
|------------|--------|
| {페이지 제목} | [열기]({페이지 URL}) |

---

**답변 작성 원칙:**
- 위키에 없는 내용은 추측하지 않는다 — "문서에서 찾을 수 없습니다"라고 명시
- 여러 페이지에서 정보를 합칠 경우 각 출처를 모두 표시
- 코드, 설정값, 명령어는 그대로 인용
- 일본어 번역은 자연스러운 비즈니스 일본어 사용

### 4단계: 추가 질문 확인

`AskUserQuestion`으로 확인:
- "추가로 궁금한 것이 있나요? / 他に質問はありますか？"
- 옵션: 추가 질문 있음 / 없음 (종료)

추가 질문이 있으면 **2단계부터 반복** (Space key는 이전 것 유지).

## 오류 처리

- **검색 결과 0건**: "해당 Space에서 관련 내용을 찾지 못했습니다. 키워드를 바꿔 다시 시도해 주세요."
- **MCP 연결 실패**: 오류 메시지를 사용자에게 그대로 전달하고 재시도 여부 확인
- **페이지 접근 권한 없음**: 접근 가능한 결과만 사용하고 권한 문제 안내
