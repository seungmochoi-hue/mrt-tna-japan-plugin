# Gmail Triage 스킬

받은편지함의 읽지 않은 메일(최근 7일)을 4단계로 분류하고, Gmail 라벨을 적용한 뒤 Slack DM으로 요약을 보낸다.

## 분류 기준

| 등급 | 라벨 | 판단 기준 |
|------|------|----------|
| 🔴 즉시회신 | `inbox-triage/reply-urgent` | 오늘 내 답장 필요 — 마감일 언급, 승인/결정 요청, 직접 질문, "급함"·"urgent"·"ASAP" 포함 |
| 🟡 회신필요 | `inbox-triage/reply-normal` | 답장이 필요하나 긴급하지 않음 — 질의응답, 일정 조율, 검토 요청 |
| 🔵 확인필요 | `inbox-triage/read-only` | 읽어야 하나 회신 불필요 — 공지, 리포트, 공유 문서, CC 수신 |
| ⚪ FYI | `inbox-triage/fyi` | 실질적 액션 불필요 — 뉴스레터, 자동 알림, 시스템 메일, 광고 |

**판단 원칙:**
- 제목과 발신자만으로 판단하기 어려우면 본문 첫 200자를 확인한다.
- 동일 스레드가 여러 건이면 가장 최신 메일 기준으로 분류한다.
- 불확실할 때는 한 단계 높은 등급으로 분류한다 (보수적 판단).

## 실행 순서

### 1단계: 읽지 않은 메일 수집

Google MCP 툴로 최근 7일치 읽지 않은 메일 목록을 가져온다.

```
검색 쿼리: is:unread newer_than:7d in:inbox
```

가져올 필드: messageId, subject, from, date, snippet (본문 미리보기)

메일이 50건 초과라면 먼저 건수를 사용자에게 알리고 계속 진행한다.

### 2단계: 분류

각 메일을 위 기준에 따라 4단계 중 하나로 분류한다.
분류 결과를 내부적으로 리스트로 관리한다:

```
{
  messageId: "...",
  subject: "...",
  from: "...",
  date: "...",
  category: "reply-urgent" | "reply-normal" | "read-only" | "fyi",
  reason: "분류 근거 한 줄"
}
```

### 3단계: Gmail 라벨 적용

분류된 각 메일에 해당 라벨을 적용한다.

- 라벨이 없으면 먼저 생성한다 (`inbox-triage/reply-urgent` 등 4개).
- 이전 `inbox-triage/*` 라벨이 붙어 있으면 제거 후 새 라벨 적용 (재실행 시 중복 방지).
- 라벨 적용 실패 건은 건너뛰고 계속 진행하며, 마지막에 실패 건수를 표시한다.

### 4단계: Slack 채널 확인 및 승인

메시지를 보내기 전에 반드시 사용자에게 전송 대상을 확인한다.

분류 결과 요약을 터미널에 먼저 출력한 뒤, 다음과 같이 질문한다:

```
📬 분류 완료: 🔴 즉시회신 N건 | 🟡 회신필요 N건 | 🔵 확인필요 N건 | ⚪ FYI N건

Slack 메시지를 어디로 보낼까요?
  1. 나에게 DM
  2. 채널 직접 입력 (예: #inbox-digest, @승모)
  3. 발송 취소
```

사용자가 선택하거나 채널명을 입력하면 해당 대상으로 전송한다.
"취소" 또는 3을 선택하면 Slack 발송 없이 Gmail 라벨 적용 결과만 보고한다.

### 5단계: Slack 메시지 발송

`mcp__plugin_slack_slack__slack_send_message` 툴로 승인된 채널/DM에 메시지를 보낸다.

**메시지 형식:**

```
📬 *Gmail Triage 완료* — {오늘 날짜}
읽지 않은 메일 {총N}건 분류 완료

🔴 즉시회신 {N}건 | 🟡 회신필요 {N}건 | 🔵 확인필요 {N}건 | ⚪ FYI {N}건

━━━━━━━━━━━━━━━━━━━━
🔴 *즉시회신 필요*
• [{발신자}] {제목} — {분류 근거}
• [{발신자}] {제목} — {분류 근거}
(즉시회신이 없으면 이 섹션 생략)

🟡 *회신 필요*
• [{발신자}] {제목}
• [{발신자}] {제목}
(5건 초과 시 상위 5건만 표시 후 "외 N건" 표기)

━━━━━━━━━━━━━━━━━━━━
✅ Gmail 라벨 적용 완료
```

FYI와 확인필요는 Slack 메시지에 개수만 표시하고 목록은 생략한다 (노이즈 방지).

### 6단계: 완료 보고

터미널에 간단한 요약을 출력한다:
- 총 처리 건수, 각 카테고리별 건수
- 라벨 적용 실패 건수 (있는 경우)
- Slack DM 발송 성공 여부

## 오류 처리

- **Google MCP 연결 실패**: 오류 메시지를 출력하고 중단. `claude mcp list`로 연결 상태 확인을 안내한다.
- **Slack 발송 실패**: Gmail 라벨은 이미 적용됐으므로 터미널에 결과를 출력하고 Slack 실패만 알린다.
- **메일 없음**: "최근 7일 내 읽지 않은 메일이 없습니다"를 출력하고 종료한다.

## 사용 예시

사용자가 다음 중 하나를 말하면 이 스킬을 실행한다:
- "메일 정리해줘"
- "Gmail 트리아지 돌려줘"
- "안 읽은 메일 분류해줘"
- "받은편지함 정리"
- "inbox triage"
